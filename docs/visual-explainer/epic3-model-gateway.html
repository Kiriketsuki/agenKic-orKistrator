Tier system. Tasks are classified into three tiers: cheap (Haiku — simple lookups, formatting), mid (Sonnet — code generation, analysis), and frontier (Opus — complex architecture, novel problems). The judge model itself runs on the cheapest tier, costing fractions of a cent per classification.
02

Components

T1 — Gateway Interface & Types
The Contract

Core interfaces (Gateway, Router, Completer, CostTracker) and value types (TaskSpec, RoutingDecision, CompletionRequest/Response, CostRecord). All types are immutable by convention.

  • gateway/gateway.go — 147 lines, 12 types
  • gateway/errors.go — 93 lines, 7 sentinel errors
T2/T3 — LiteLLM Client & Adapters
Provider Layer

OpenAI-compatible HTTP client for the LiteLLM proxy. Per-provider FormatAdapters handle quirks: Anthropic system prompts, OpenAI model prefixes, Ollama name stripping. Registry resolves model names to adapters at request time.

  • gateway/litellm.go — LiteLLM HTTP client (210 lines)
  • providers/provider.go — Registry + FormatAdapter interface
  • providers/anthropic.go, openai.go, ollama.go
T4 — Judge-Router
Complexity Classifier

Haiku classifies task descriptions into cheap/mid/frontier tiers. Single-word response parsing. Falls back to mid on ambiguous output. Override via TaskSpec.OverrideTier bypasses classification entirely.

  • gateway/router.go — 165 lines
  • Classification prompt: 3-tier, one-word response
  • Functional options: WithJudgeModel, WithDefaultTier
T5/T6 — Config & Fallback Chains
YAML Config + Resilience

Loads config/models.yaml with tier definitions, provider endpoints, model assignments, and pricing tables. FallbackCompleter wraps per-model completers and tries the chain on failure. Multi-error unwrapping via errors.Is.

  • gateway/config.go — YAML loader + validation (124 lines)
  • gateway/fallback.go — FallbackCompleter (112 lines)
  • config/models.yaml — 3 tiers, 3 providers, pricing
T7 — Cost Tracking
Token Economics

Per-request cost recording with EstimateCost using pricing tables. InMemoryCostTracker (thread-safe, spike-phase) stores records and generates CostReports with per-tier aggregation over half-open time periods. Production will use Redis-backed storage.

  • gateway/cost.go — 109 lines, 3 Claude model prices
  • NewCostRecordFromResponse bridges completion → cost
  • Report: period filtering, tier grouping, totals
Integration
Orchestrator Wiring

Supervisor exposes gateway via CompletionRegistry. BlockingSubmitter bridges DAG task submission with completion awaiting. gRPC handlers in ipc/handlers.go wrap gateway operations for external callers.

  • supervisor/completion.go — registry + awaiter
  • dag/blockingsubmitter.go — DAG → completion bridge
  • ipc/handlers.go — gRPC service methods
