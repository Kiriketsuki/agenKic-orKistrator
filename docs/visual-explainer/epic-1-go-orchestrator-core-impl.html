ipc (gRPC Server)

internal/ipc/

OrchestratorServer with 7 RPCs. Routes to supervisor, store, or DAG engine based on operation type. Includes gRPC health and HTTP health endpoints.

supervisor

internal/supervisor/

Agent pool manager. Per-agent mutexes serialize compound operations. Runs heartbeat monitoring and task assignment loops. Exponential backoff on store errors.

agent (State Machine)

internal/agent/

Pure state machine: Machine has no mutable state. All state lives in StateStore. Uses CompareAndSetAgentState (CAS) for optimistic locking on transitions.

dag (DAG Engine)

internal/dag/

Validates DAGSpec, builds Graph, runs Kahn's topological sort, then level-by-level parallel fan-out. Fail-fast: context cancel on first error.

state (StateStore)

internal/state/

16-method interface: agent CRUD, CAS transitions, event stream (pub/sub/ack), task queue (sorted set), health ping. MockStore + Redis implementations.

health (Aggregator)

internal/health/

Batch-reads agent states (GetAllAgentStates), queue length, Redis ping, and DAG active count. Produces an immutable HealthSnapshot. Readiness = Redis up + agents registered.

02 Agent State Machine

Four states, five events, one reset edge. The transition table is a pure function (ValidTransition) with no side effects. Machine.ApplyEvent reads state, validates, then writes atomically via CAS. 

stateDiagram-v2
  direction LR
  [*] --> Idle
  Idle --> Assigned : EventTaskAssigned
  Assigned --> Working : EventWorkStarted
  Working --> Reporting : EventOutputReady
  Reporting --> Idle : EventOutputDelivered

  Assigned --> Idle : EventAgentFailed
  Working --> Idle : EventAgentFailed
  Reporting --> Idle : EventAgentFailed
Optimistic Locking via CAS: Machine.ApplyEvent calls GetAgentState, then CompareAndSetAgentState(agentID, expected, next). If another goroutine changed the state between read and write, a *StateConflictError is returned and the caller retries or re-enqueues. This replaced per-agent mutex locking for state transitions while the supervisor mutex remains for compound operations (state transition + field writes).

Parallel within levels

Each node in a level runs in its own goroutine. A sync.WaitGroup blocks until the entire level completes before advancing. Nodes A and B execute concurrently; C and D wait for their predecessors.

Fail-fast on error

If any node fails, its goroutine calls cancel() on the shared context. Remaining nodes in the current level see ctx.Err(); all future levels are marked "skipped: upstream failure".

MVP Limitation (StoreSubmitter): The StoreSubmitter is an enqueue-only adapter. SubmitTask returns nil on enqueue success, not on task completion by an agent. DAG dependency ordering is structural, not execution-verified. A production TaskSubmitter from the model gateway epic must block until the agent finishes.

04 gRPC Service Layer

Seven RPCs defined in proto/orchestrator.proto, routed by operation type. The server holds references to Supervisor, StateStore, and DAGEngine (an interface satisfied by dag.Executor).

05 Supervisor

Manages the agent pool via two background loops: heartbeat monitoring and task assignment. Per-agent mutexes serialize compound operations; CAS handles atomic state transitions.

Heartbeat Loop

Ticks every 5s (configurable). Lists agents, reads LastHeartbeat, crashes non-idle agents whose heartbeat exceeds 30s staleness threshold.

Task Assignment Loop

Ticks every 100ms. Dequeues highest-priority task, finds idle agent (skipping cooldown/circuit-open), acquires per-agent mutex, CAS transitions to Assigned, persists CurrentTaskID.

Restart Policy

Exponential backoff: base * 2^(n-1) capped at 30s. Circuit breaker: >5 crashes within 60s window opens circuit. RecordSuccess resets everything.

TOCTOU Guard: A cooldown sentinel is pre-populated before the CAS transition so findIdleAgent cannot observe the agent as IDLE without a cooldown entry. If ApplyEvent reveals the agent was already IDLE (spurious crash from concurrent completion), the sentinel is cleaned up without recording a crash.

07 Known MVP Limitations

Four tracked deferrals with GitHub issue numbers. These are intentional scope cuts for Epic 1, with clear integration points for future epics.

#71
Heartbeat RPC
No agent-initiated heartbeat RPC. The supervisor reads LastHeartbeat from the store, but agents cannot actively push heartbeats. Requires agent-initiated RPCs (bidirectional).
#72
DAG Completion Tracking
StoreSubmitter is enqueue-only. MarkNodeCompleted fires on enqueue success, not task execution. Real completion requires model gateway's TaskSubmitter to block until agent finishes.
#73
Agent-Initiated RPCs
Agents cannot call back to the orchestrator (heartbeat, work-started, output-ready). Currently the supervisor drives all transitions. Requires reverse-RPC or polling pattern.
#74
StreamOutput Business Logic
Bidi streaming is wired but only echoes acknowledgements. No output storage, aggregation, or routing. Depends on model gateway and terminal substrate integration.
