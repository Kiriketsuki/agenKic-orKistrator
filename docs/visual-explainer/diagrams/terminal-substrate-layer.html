Validation Guards

GUARD

ValidateSessionName

Rejects empty strings and names containing characters outside [a-zA-Z0-9_.\-]. Prevents tmux interpretation of special characters in session targets. Guards 5 of 6 interface methods.

tmux_errors.go:80

GUARD

ValidateCommand

Rejects null bytes (\x00), dangerous control characters (\x01-\x1f excluding tab/newline/CR), and commands exceeding MaxCommandLen (8192). Structural safety on execution interface.

Supervisor Integration

LIFECYCLE

Terminal Session Lifecycle

  1. Init: main.go:48 — NewTmuxSubstrate() checks tmux on PATH. Falls back to headless if unavailable.
  2. Wire: main.go:52 — WithSubstrate(sub) injects via functional option. Nil-safe throughout.
  3. Spawn: supervisor.go:121-126 — RegisterAgent calls SpawnSession("agent-"+id, ""). Failure logged, agent continues headless.
  4. Crash: supervisor.go:293-298 — crashAgent calls DestroySession for the crashed agent's session.
  5. Shutdown: supervisor.go:135-151 — Stop() iterates all agents, destroys each session with context.Background().
  6. Ordering: main.go:96-102 — cancel() → GracefulStop() → httpHealth.Shutdown() → sv.Stop()
Graceful degradation. The substrate field is typed as terminal.Substrate (nullable). Every usage is guarded with if sv.substrate != nil. A missing or failed terminal layer never prevents an agent from registering, processing tasks, or shutting down cleanly.
