04

FallbackCompleter: tryChain Logic

The core fallback loop iterates through [primary, fallback-1, fallback-2, ...], returning on the first success. A context guard at the top of each iteration provides cancellation-aware abort with accumulated error preservation.

76func (fc *FallbackCompleter) tryChain(ctx context.Context, req CompletionRequest, chain []string) (CompletionResponse, error) { 77 var errs []ProviderError 78 79 for i, model := range chain { 80 if err := ctx.Err(); err != nil { // ← ctx guard 81 errs = append(errs, ProviderError{Provider: model, Err: err}) 82 return CompletionResponse{}, &FallbackError{Errors: errs} 83 } 84 85 c, ok := fc.completers[model] // ← completer lookup 86 if !ok { 87 errs = append(errs, ProviderError{...}) 88 continue // ← skip, try next 89 } 90 96 resp, err := c.Complete(ctx, r) // ← actual provider call 97 if err != nil { ... continue } // ← record error, try next 107 resp.FallbackUsed = i > 0 // ← true only on fallback 108 return resp, nil 109 } 111 return CompletionResponse{}, &FallbackError{Errors: errs} 112}
Context Guard
Fires at the top of each iteration. At i=0: catches pre-cancelled contexts. At i>0: stops the chain after a slow provider if the context timed out during the call. Preserves all accumulated errs from prior iterations.

fallback.go:80-83
Graceful Skip
Missing completer (no registered provider for model) appends ErrNoProvider and continues. This handles empty-string model names from malformed YAML gracefully — no crash, chain continues.

fallback.go:85-92
Error Recording
If the provider returns a *ProviderError, it's recorded verbatim. Generic errors are wrapped with c.Provider(). Ordered slice position maps to chain position for disambiguation.

fallback.go:97-104
05

Error Hierarchy & Context Transparency

FallbackError.Unwrap() returns []error (Go 1.20+ multi-error), surfacing both ErrAllProvidersFailed and any context.Canceled / context.DeadlineExceeded found in the chain. This enables errors.Is(err, context.Canceled) to work transparently.

ERROR TREE
*FallbackError
Aggregates all chain failures
ErrAllProvidersFailed
Sentinel — always returned via Unwrap
context.Canceled
Returned when chain stopped due to ctx cancellation
context.DeadlineExceeded
Returned when chain stopped due to ctx timeout
[]ProviderError
Individual per-attempt errors (inspect via fe.Errors)
UNWRAP IMPLEMENTATION
func (e *FallbackError) Unwrap() []error { errs := []error{ErrAllProvidersFailed} for _, pe := range e.Errors { if pe.Err == context.Canceled || pe.Err == context.DeadlineExceeded { errs = append(errs, pe.Err) break } } return errs }
Council Fix

Prior design returned error (single). Now returns []error enabling errors.Is(err, context.Canceled) without manual fe.Errors iteration. Commit 0c6bab0.
