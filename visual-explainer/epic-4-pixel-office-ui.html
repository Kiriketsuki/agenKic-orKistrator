Pixel Office UI

A Godot 4 desktop application rendering a top-down pixel-art office where each AI agent appears as an animated sprite character at their desk. Terminal output streams live via godot-xterm. Agent state drives sprite animations. Panel layouts are dockable and persistent.

Agent Sprite State Machine

🧍 IDLE idle_loop.anim
TaskAssigned →
📋 ASSIGNED stand_up.anim
WorkStarted →
⌨️ WORKING typing.anim
OutputReady →
📤 REPORTING celebrate.anim
OutputDelivered →
🧍 IDLE idle_loop.anim
EventAgentFailed → resets to Idle from any state
GDScript state machine. agent_character.gd receives agent_state_changed signals emitted by the gRPC bridge GDExtension. Each transition calls AnimationPlayer.play() with the matching clip. States map 1:1 to the orchestrator's Go agent state machine via the GetAgentState gRPC stream.

Data Flow & API Contract

01

Connect

GDExtension bridge opens a gRPC channel to the Go Orchestrator on startup. Reconnects with exponential backoff on disconnect.

02

Register

Calls GetAgentState stream for each agent. Bridge emits agent_registered signal. Office scene spawns agent sprite at desk.

03

State Events

Orchestrator pushes state transitions via stream. Bridge emits agent_state_changed(id, new_state). Agent scene transitions animation.

04

Terminal

Bridge streams TerminalOutput chunks from the orchestrator's tmux capture. terminal_manager.gd feeds bytes into the godot-xterm node.

05

Overlay

On agent click, overlay subscribes to the agent's state stream for live name, task, state, and uptime. Unsubscribes on dismiss.

API contract. The Godot app is a pure consumer — it calls RegisterAgent, GetAgentState (server-streaming), and a terminal output stream (TBD in orchestrator proto). It never writes agent state; all mutations happen inside the Go orchestrator. The GDExtension bridge translates gRPC proto messages into Godot Dictionary objects delivered via signals.

Risks

Risk Likelihood Mitigation
godot-xterm PTY is Linux/macOS only — Windows gets display-only High Document limitation clearly. Implement SSH output bridge as Windows fallback in a later iteration.
godot-xterm has no stable v1.0 release Medium Pin to a specific commit in the submodule. Wrap terminal access behind an abstraction layer so the implementation can be swapped.
gRPC client in GDScript is non-trivial Medium Use GDExtension (C++) for the gRPC client. Expose a simple signal API to GDScript. HTTP/REST bridge is a fallback if GDExtension build is blocked.
Pixel-perfect rendering breaks at non-integer DPI Medium Force integer scaling in project.godot. Test at 1080p, 1440p, and 4K. Document supported resolution list.
